Scraping the Web, One Site at a Time

From building a movie database with rankings and reviews, to auto-generating Spotify playlists and tracking Amazon prices in real time — these projects turn scattered web data into something actually useful.

From API to App

This project started as a simple question: what if I could build a website around my own taste in movies? Using Flask and SQLAlchemy, I built a full database-driven app that pulls real movie data from an external API and lets me rank, rate, and review my all-time favorites — turning a static list into something dynamic and personal.

  • Connected to The Movie Database (TMDB) API to pull real movie data — titles, release dates, descriptions, and poster images.
  • Designed a SQLAlchemy database model to store and manage rankings, ratings, and personal reviews for each movie.
  • Built full CRUD functionality — adding, editing, updating, and deleting movies dynamically through Flask routes.
  • Used WTForms to handle and validate user input for ratings and reviews before saving to the database.
  • Parsed and cleaned JSON API responses (like extracting just the year from a full release date) to keep the data structured and usable.

The Nostalgia Engine

What if you could relive any week in music history with one click? This project lets you pick a date, scrapes that week's Billboard Hot 100 chart, and automatically builds a matching playlist straight into my Spotify account — turning chart history into something I can actually listen to.

  • Took a user-input date and scraped that exact week's Billboard Hot 100 chart using BeautifulSoup.
  • Parsed song titles and artist names from the page's HTML structure, cleaning and pairing each track with its correct artist.
  • Authenticated with the Spotify API using Spotipy and OAuth to securely access my account.
  • Searched Spotify for each scraped track and retrieved its unique track ID, handling missing matches gracefully.
  • Automatically added every matched song directly into a real Spotify playlist — no manual searching required.

Set It and Forget It

Why keep refreshing a product page hoping for a sale? This script scrapes a live Amazon listing, checks the current price against a target threshold, and automatically emails me the moment it's worth buying — turning manual price-watching into a fully automated alert system.

  • Scraped a live Amazon product page using BeautifulSoup to extract the current price and product name in real time.
  • Set a custom price threshold, so the script only acts when the item actually drops to a worthwhile price.
  • Connected to Gmail's SMTP server to send automated email alerts the moment a price drop is detected.
  • Secured sensitive credentials like email and password using environment variables instead of hardcoding them.
  • Built the alert message dynamically with an f-string, pulling in the live product name and price for a personalized notification.
